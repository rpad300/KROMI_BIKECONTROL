# Ride Page v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development

**Goal:** Rewrite ride.html as a world-class editorial page with split architecture, club theming, dark/light mode, SEO, live tracking, and full responsiveness.

**Architecture:** Thin HTML template + external CSS + external JS engine. Club theme from `clubs.theme` jsonb column.

**Spec:** `docs/superpowers/specs/2026-04-23-ride-page-v2-design.md`

---

## Tasks

### Task 1: DB migration — clubs.theme column
- Add `theme jsonb` to clubs table
- Backfill existing clubs with null (uses KROMI defaults)

### Task 2: ClubBackofficeTab — Theme section
- Add "Tema" section in ClubBackofficeTab.tsx
- Fields: primary colour, secondary colour, heading font (dropdown), body font (dropdown)
- Save to `clubs.theme` via updateClub

### Task 3: ride-styles.css — Complete stylesheet
- Dark theme (default) + Light theme via [data-theme] and prefers-color-scheme
- All sections: hero, stats, map, altimetry, segments, timeline, weather, POIs, riders, gallery, footer
- Responsive: mobile (<640), tablet (640-1024), desktop (>1024)
- Print, reduced-motion, accessibility
- Club theme CSS custom properties overridable

### Task 4: ride-engine.js — Complete JS engine
- All current logic extracted from ride.html into external file
- Data fetch, GPX parse (namespace strip), profile builder, D+ smoother
- Speed model (FIT-calibrated bands + group penalty)
- Render: hero, stats, map (Google Maps), altimetry SVG, segments + mini-maps
- Render: chronogram (smart stops, KPI cards), weather (Open-Meteo), POIs (Google Places)
- Render: riders table, photo gallery + lightbox
- SEO: inject JSON-LD + update OG meta dynamically
- Live tracking: check status, show badge + sticky bar
- Dark/light toggle with localStorage persistence
- Club theme application from clubs.theme

### Task 5: ride.html — Thin template rewrite
- Minimal HTML with semantic structure
- Loads: ride-styles.css, Google Fonts, Google Maps (async), ride-engine.js
- Section containers with IDs (hidden by default, shown by engine)
- No inline styles, no inline JS logic

### Task 6: Build verification + deploy
- npm run build (verify CSS/JS copied to dist)
- Verify navigateFallbackDenylist includes ride.html
- Push + Vercel auto-deploy
